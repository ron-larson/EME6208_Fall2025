Screencasting

This project required presenting a software demonstration topic lasting approximately 2 – 4 minutes by using either a trial version of Camtasia, or CamStudio, or some other equivalent tool to record a narrated screencast. A follow-on project had similar requirements, but instead focused on reviewing and evaluating an AI video production tool of my choice.

Software-Demo Screencast

How to Debug in Scratch

Project Reflection

For the software demo screencast, I once again selected a specific, focused skill from my 7th-grade "Block Busters" unit plan: debugging a common sprite movement error in Scratch.

A two-column script was drafted before recording, mapping the narration directly to the on-screen actions and planning cues for post-production effects, to help ensure instructional clarity.

Tool selection was critical. After evaluating options, I chose the desktop version of Camtasia (trial) as it was the only tool that provided the necessary post-production zoom, pan, and cursor animation features required by the rubric.

The production process involved recording, followed by a detailed editing phase. In editing, I added the planned zoom and pan animations, applied cursor highlighting, and polished the video with a solid background and added title/end frames that used fade transitions. The final video (2:32) was then exported as an .mp4 and uploaded to YouTube.
