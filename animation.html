Project details for my animated sequence.

What is Debugging?

For this project, I created a 14-second animation titled "What is Debugging?" to visually explain a core concept from my 7th-grade computer science unit, "Block Busters: Building and Fixing Our First App." The animation's educational value is in how it demystifies debugging for students. It visually portrays a common bug—a sprite moving through a wall—and contrasts a vague instruction ("MOVE 100 STEPS") with a precise, logical fix ("IF NOT touching_wall THEN MOVE"). This reinforces the idea from our unit's "Debugging Log" that fixing errors is not a random guess, but a systematic process of testing, identifying a logical flaw, and implementing a specific solution.

My creation process began with storyboarding the animation in Adobe Animate. I imported a Cat vector sprite image from the Scratch coding environment (because it is used in a lot of beginning K-12 computer coding classes) and drew the simple wall rectangle directly on the stage. The primary animation was achieved using a motion tween to slide the cat. I also used motion tweens for all transition effects, applying them to the Alpha property of the title text, the pseudocode, the red "X" for the error, and the green "✓" for the solution. After exporting this visual-only .mp4 file, I imported it into Canva to sync it with my audio voice-over that I recorded using Audacity, creating the final video.

The main challenge I faced was the complexity of Adobe Animate. As a professional-grade tool, its learning curve is steep for a short project. For instance, creating the fade effects required first converting each text block and shape into a 'symbol' before a motion tween could even be applied. It then took time to locate the Alpha property slider within the object's properties to control the transparency. The most significant challenge, however, was mastering the timeline to precisely synchronize all these layered tweens. Getting the cat's movement, the text, and the "X" and "✓" symbols to appear and disappear at the exact moments to match the 14-second voice-over required several attempts and fine-tuning.
