About the App

The "Block Busters" Unit Quiz App is a multiple-choice assessment tool designed for a 7th-grade Computer Science unit plan entitled "Block Busters: Building and Fixing Our First App". It cycles through 18 questions covering core unit concepts like Algorithms, Debugging, and Collaboration.

Project Reflection: Engineering the "Block Busters" Unit Quiz

For Project 9, I customized the "QuizMe" tutorial to engineer the "Block Busters" Unit Quiz, a mobile application designed as a summative review tool for my previously developed 7th-grade Computer Science unit plan. The app utilizes parallel lists to manage 20 targeted questions, specifically tailored to reference class artifacts like the "Marshmallow Challenge" and the "PB&J Algorithm" video. This ensures the digital tool feels relevant to the physical classroom experience.

Development & Enhancements:
My development process utilized a hybrid workflow, coding on a Windows 11 PC while live-testing on an Android phone via the AI Companion app. To exceed the basic requirements, I focused on User Experience (UX) and Visual Design. I integrated a custom app icon and used GIMP to process a background image at 38% opacity for optimal text readability. I also hardened the logic by scripting a custom ToggleButtons procedure to prevent a "spam-click" bug I discovered during testing, ensuring the scoring system remains robust. Specifically, I used a control variable to disable the answer buttons immediately after a user selection, preventing them from racking up points by repeatedly tapping the correct answer before the 'Next' button appeared. This small logic gate was essential for maintaining the integrity of the assessment data.

Evaluating App Inventor:
From an educator's perspective, App Inventor is an excellent parallel to the Scratch environment, perfectly bridging the gap between "drag-and-drop" logic and real-world deployment. However, coming from a background in C++ and Python, I found the "Blocks" editor occasionally restrictive. Logic structures I could type in seconds required dragging, dropping, and snapping multiple visual elements. While excellent for novices, it highlighted the efficiency trade-offs of visual programming.

Next Steps:
Currently, the quiz is sequential. My goal for "Version 2.0" is to implement question randomization, which requires a complex "list of lists" data structure to maintain data integrity—a significant logical challenge compared to standard text-based arrays. Beyond this tool, I plan to explore Android Studio using Kotlin to leverage my object-oriented programming background, mirroring the progression from "Blocks to Text" that I model for my students.
