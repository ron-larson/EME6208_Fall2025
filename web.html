Project Reflection

For my web design work, I focused on exploring different methods for creating interactive assessments. I adapted two separate projects to build learning tools for a 7th-grade computer science unit that I previously developed, "Block Busters: Building and Fixing Our First App", with each project presenting a unique set of technical challenges and learning opportunities.

The first project, the "Code Breakers" Quiz, was an exercise in JavaScript logic. I modified a foundational script from JavaScript Kit to expand the question bank from five to ten, creating new, age-appropriate content on the topic of algorithms. The primary challenge was a data-parsing bug; I discovered that double quotes (") within the question text caused a JavaScript error that prevented questions from rendering. I debugged this by systematically re-adding each question, isolating the error, and implementing the correct escape sequence (\").

The second project, the "Code Concepts" Drag-and-Drop Assessment, was a deeper dive into layout, CSS, and JavaScript-driven DOM manipulation. I significantly re-engineered the structure from three to four content categories (Algorithm, Variable, Loop, and Conditional) and replaced all content items. To enhance its instructional use, I added a "Quiz Mode" checkbox that dynamically instructs the JavaScript to lock answers upon being dropped. The main challenge was a series of CSS and layout conflicts. I had to widen the main containers to prevent the fourth box from wrapping, and then debug a script that was overriding my styles. I solved this by commenting out a problematic JavaScript height-fix and using CSS (min-height: 150px; and overflow: auto) to manage the layout, fix a collapsing container, and ensure drop-zones were visible.

Finally, these projects highlighted the importance of User Interface (UI) usability. In the drag-and-drop activity, I realized that simply having the code "work" was not enough; the user needed visual feedback. I added a cursor change (cursor: move) in the CSS so students would intuitively know which elements were interactable. This small detail significantly improved the user experience, reinforcing the lesson that good web design is as much about how a tool feels as how it functions. This work tracked my own tool progression as well; I edited the first project in Notepad, but as the complexity grew, I moved to Visual Studio Code, which proved invaluable for managing the interconnected HTML, CSS, and JavaScript files.
